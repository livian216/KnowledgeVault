# ZeroTier 远程访问使用教程：Windows 笔记本远程连接 Ubuntu 实验室服务器

本文档用于记录 ZeroTier 的基础使用方法，适用于如下典型场景：

```text
外地 Windows 笔记本  ←→  实验室 Ubuntu 服务器
```

目标是：人在外地时，可以通过 ZeroTier 虚拟内网 IP，直接 SSH 连接实验室 Ubuntu 服务器，不需要公网 IP，不需要路由器端口映射，也通常不需要额外租用阿里云、腾讯云等公网云服务器。

---

## 1. ZeroTier 能解决什么问题

ZeroTier 可以把分散在不同网络环境下的设备加入同一个虚拟局域网。比如外地笔记本和实验室服务器本来不在同一个局域网，也没有公网 IP，但只要它们都安装 ZeroTier、加入同一个 ZeroTier Network，并在后台授权，就可以像处在同一个局域网一样互相访问。

本文档主要覆盖：

```text
1. ZeroTier 网络创建
2. Windows 系统安装与加入网络
3. Ubuntu 系统安装与加入网络
4. 后台授权与 IP 确认
5. Windows 远程 SSH 连接 Ubuntu
6. 常见问题排查
```

官方参考资料：

- ZeroTier Quickstart: https://docs.zerotier.com/quickstart/
- ZeroTier CLI 文档: https://docs.zerotier.com/cli/
- ZeroTier 下载页面: https://www.zerotier.com/download/
- ZeroTier 定价页面: https://www.zerotier.com/pricing/

## 2. 整体网络结构

假设你的使用场景如下：

```text
外地 Windows 笔记本
    |
    | 互联网
    |
ZeroTier 虚拟局域网
    |
    | 互联网
    |
实验室 Ubuntu 服务器
```

ZeroTier 会给两台设备分别分配一个虚拟内网 IP，例如：

```text
Windows 笔记本 ZeroTier IP：10.10.10.10
Ubuntu 服务器 ZeroTier IP：10.10.10.23
```

之后你在 Windows 上远程连接 Ubuntu 时，不再使用实验室局域网 IP，而是使用 Ubuntu 的 ZeroTier IP：

```powershell
ssh your-username@10.10.10.23
```

其中 `your-username` 需要替换成你 Ubuntu 服务器上的实际用户名。

## 3. 是否需要租用公网云服务器

对于“外地笔记本远程实验室服务器”这个基础场景，通常不需要租用阿里云、腾讯云、华为云等公网云服务器。

你只需要满足：

```text
1. 外地笔记本可以正常上网；
2. 实验室服务器可以正常上网；
3. 两台设备都安装 ZeroTier；
4. 两台设备加入同一个 ZeroTier Network；
5. 在 ZeroTier 后台授权两台设备；
6. Ubuntu 服务器开启 SSH 服务。
```

也就是说，基础使用不需要：

```text
公网 IP
端口映射
DDNS
阿里云中转服务器
实验室路由器管理员权限
```

只有在以下情况下，才可能考虑额外租用公网云服务器作为中转或辅助节点：

```text
ZeroTier 一直无法直连；
连接长期不稳定；
SSH 延迟明显偏高；
远程桌面或大文件传输速度很慢；
实验室网络策略非常严格；
你希望完全掌控中继节点和网络路径。
```

对于初学和日常 SSH 远程管理，建议先使用 ZeroTier 官方默认方案，不要一开始就引入云服务器中转。

## 4. ZeroTier 是否需要付费

对于个人远程连接少量设备的场景，ZeroTier 免费版通常足够使用。

以本文档场景为例：

```text
1 台外地 Windows 笔记本
1 台实验室 Ubuntu 服务器
```

总共只有 2 台设备，属于非常轻量的个人使用场景。ZeroTier 官方免费版通常已经能够满足这种基础远程连接需求。

需要注意的是，ZeroTier 的免费额度、套餐规则可能会随时间调整。如果以后需要接入大量设备，例如多台开发板、NAS、摄像头、地面站电脑、服务器等，建议重新查看官方定价页面：

```text
https://www.zerotier.com/pricing/
```


### 4.1 免费版一个网络能否接入多地多台设备

可以。ZeroTier 免费版即使只能创建一个 Network，这个 Network 也不是指现实中的某一个物理局域网，而是指一个由 ZeroTier 创建的虚拟局域网。这个虚拟网络可以同时接入分布在不同城市、不同运营商、不同路由器、不同局域网下的多台设备。

例如你有三台设备：

```text
北京设备
杭州设备
出差随身携带的笔记本
```

只要这三台设备都满足以下条件，就可以加入同一个 ZeroTier Network：

```text
1. 每台设备都可以正常访问互联网；
2. 每台设备都安装了 ZeroTier One；
3. 每台设备都加入同一个 Network ID；
4. 每台设备都在 ZeroTier Central 后台完成授权；
5. 设备数量没有超过当前账户免费额度。
```

加入同一个 Network 后，ZeroTier 会给每台设备分配一个虚拟内网 IP。例如：

```text
北京设备 ZeroTier IP：10.10.10.11
杭州设备 ZeroTier IP：10.10.10.12
出差笔记本 ZeroTier IP：10.10.10.13
```

此时三台设备虽然处于不同城市、不同现实局域网，但在 ZeroTier 看来，它们都属于同一个虚拟局域网。因此，出差笔记本可以直接访问北京设备和杭州设备：

```powershell
ssh 用户名@北京设备的ZeroTier_IP
ssh 用户名@杭州设备的ZeroTier_IP
```

北京设备和杭州设备之间也可以通过对方的 ZeroTier IP 互相访问：

```bash
ssh 用户名@对方设备的ZeroTier_IP
```

需要特别注意的是：免费版限制的是账户下可用的网络数量和设备数量，不是限制设备必须位于同一个物理地点。也就是说，一个 ZeroTier Network 可以把北京、杭州、出差笔记本等不同地区的设备统一纳入同一个虚拟内网。

对于个人使用来说，三台设备通常属于免费额度内的轻量使用场景。但是 ZeroTier 官方免费额度、套餐名称和设备数量上限可能会随时间或账号类型调整，因此实际部署前建议查看 ZeroTier 官方定价页面确认当前账户额度：

```text
https://www.zerotier.com/pricing/
```

可以简单记住下面这句话：

```text
ZeroTier 免费版即使只有一个网络，也可以把多个地区的多台设备放进同一个虚拟局域网；关键限制是设备总数和账户套餐额度，而不是设备所在的城市或物理局域网。
```

## 5. 创建 ZeroTier 网络

### 5.1 注册并登录 ZeroTier Central

打开 ZeroTier Central，注册或登录账号。

ZeroTier Central 入口：

```text
https://my.zerotier.com/
```

新用户一般使用新版 Central；旧用户可能仍然使用 Legacy Central。按照网页提示登录即可。

---

### 5.2 创建 Network

登录后创建一个新的 Network。

创建完成后，你会得到一个 **Network ID**，类似：

```text
你的Network_ID
```

这个 Network ID 很重要，后续 Windows 和 Ubuntu 都要用它加入同一个虚拟网络。

建议记录：

```text
ZeroTier Network ID：____________________
```

---

### 5.3 检查网络地址池

进入刚创建的 Network，找到类似 **IPv4 Auto-Assign** 或 **Managed IPs** 的配置项。

一般情况下，保持默认即可。ZeroTier 会自动给设备分配一个虚拟内网 IP。

常见地址段可能类似：

```text
10.10.10.0/24
172.22.0.0/16
192.168.191.0/24
```

新手阶段不要随意修改路由、网关、DNS、Managed Routes 等高级选项。当前目标只是让两台设备互通，不需要配置复杂路由。

---

## 6. Windows 系统安装与配置

### 6.1 下载并安装 ZeroTier One

在 Windows 笔记本上下载 ZeroTier One 的 Windows 安装包：

```text
https://www.zerotier.com/download/
```

安装时注意：

```text
1. 使用默认安装路径即可；
2. 如果系统提示安装网络驱动，选择允许；
3. 安装完成后，右下角系统托盘会出现 ZeroTier 图标。
```

---

### 6.2 Windows 加入 ZeroTier 网络：图形界面方式

在 Windows 右下角系统托盘中找到 ZeroTier 图标。

操作步骤：

```text
右键 ZeroTier 图标
→ Join New Network...
→ 输入你的 Network ID
→ 点击 Join
```

加入后，Windows 这台设备会出现在 ZeroTier Central 的成员列表中，但此时通常还不能通信，因为还没有授权。

---

### 6.3 Windows 加入 ZeroTier 网络：命令行方式

也可以使用 PowerShell 加入网络。

先以管理员身份打开 PowerShell：

```text
开始菜单
→ 搜索 PowerShell
→ 右键
→ 以管理员身份运行
```

然后执行：

```powershell
zerotier-cli join 你的Network_ID
```

例如：

```powershell
zerotier-cli join 你的Network_ID
```

如果正常，会看到类似：

```text
200 join OK
```

如果 PowerShell 找不到 `zerotier-cli`，可以优先使用系统托盘图标方式加入网络。

---

### 6.4 查看 Windows 的 ZeroTier 状态

在管理员 PowerShell 中执行：

```powershell
zerotier-cli status
```

正常情况下会看到类似：

```text
200 info xxxxxxxxxx 1.16.2 ONLINE
```

其中：

```text
ONLINE  表示 ZeroTier 客户端在线
```

也可以查看加入的网络：

```powershell
zerotier-cli listnetworks
```

如果还没有在后台授权，可能会看到：

```text
ACCESS_DENIED
```

这不是安装失败，而是说明设备还需要在 ZeroTier Central 里授权。

## 7. Ubuntu 系统安装与配置

以下步骤在实验室 Ubuntu 服务器上执行。

### 7.1 安装 curl

如果系统没有安装 `curl`，先执行：

```bash
sudo apt update
sudo apt install -y curl
```

---

### 7.2 安装 ZeroTier One

Ubuntu 可以通过官方安装脚本安装 ZeroTier：

```bash
curl -s https://install.zerotier.com | sudo bash
```

安装完成后，查看服务状态：

```bash
sudo systemctl status zerotier-one
```

如果服务没有启动，可以执行：

```bash
sudo systemctl enable zerotier-one
sudo systemctl start zerotier-one
```

---

### 7.3 Ubuntu 加入 ZeroTier 网络

执行：

```bash
sudo zerotier-cli join 你的Network_ID
```

例如：

```bash
sudo zerotier-cli join 你的Network_ID
```

正常情况下会看到：

```text
200 join OK
```

---

### 7.4 查看 Ubuntu 的 ZeroTier 状态

查看 ZeroTier 客户端是否在线：

```bash
sudo zerotier-cli status
```

正常输出类似：

```text
200 info xxxxxxxxxx 1.16.2 ONLINE
```

查看已加入的网络：

```bash
sudo zerotier-cli listnetworks
```

未授权时可能显示：

```text
ACCESS_DENIED
```

授权成功后，状态应变为：

```text
OK
```

并且会看到 ZeroTier 分配的虚拟 IP，例如：

```text
10.10.10.23/24
```

---

## 8. 在 ZeroTier 后台授权设备

这是非常关键的一步。

无论 Windows 还是 Ubuntu，加入网络后都需要在 ZeroTier Central 里授权。

操作步骤：

```text
1. 打开 ZeroTier Central；
2. 进入你创建的 Network；
3. 找到 Members / Member Devices 区域；
4. 找到刚刚加入的 Windows 笔记本；
5. 点击 Authorize 或勾选 Auth；
6. 给设备起一个易识别的名字，例如 Windows-Laptop；
7. 找到 Ubuntu 服务器；
8. 点击 Authorize 或勾选 Auth；
9. 给设备起名，例如 Lab-Ubuntu-Server。
```

建议命名方式：

```text
Windows-Laptop
Lab-Ubuntu-Server
Home-PC
Office-Server
RK3588-Board
MaixCAM-Gateway
```

授权后，ZeroTier 后台会显示每台设备的 Managed IP。请记录 Ubuntu 服务器的 ZeroTier IP：

```text
Ubuntu 服务器 ZeroTier IP：____________________
```

## 9. 测试 Windows 与 Ubuntu 是否互通

假设 Ubuntu 服务器的 ZeroTier IP 是：

```text
10.10.10.23
```

---

### 9.1 Windows ping Ubuntu

在 Windows PowerShell 中执行：

```powershell
ping 10.10.10.23
```

如果能看到类似输出，说明网络基本互通：

```text
来自 10.10.10.23 的回复: 字节=32 时间=xxms TTL=64
```

如果 ping 不通，不一定代表不能 SSH。Windows 或 Linux 防火墙可能会禁止 ICMP，但 SSH 仍可能正常。

---

### 9.2 Ubuntu ping Windows

在 Ubuntu 服务器上执行：

```bash
ping Windows的ZeroTier_IP
```

例如：

```bash
ping 10.10.10.10
```

---

### 9.3 查看 ZeroTier 网络状态

Windows 管理员 PowerShell：

```powershell
zerotier-cli listnetworks
```

Ubuntu：

```bash
sudo zerotier-cli listnetworks
```

授权成功后，状态应为：

```text
OK
```

如果仍然是：

```text
ACCESS_DENIED
```

说明后台还没有正确授权，或者授权后客户端尚未刷新状态。

## 10. 配置 Ubuntu SSH 服务

ZeroTier 只负责把网络打通。你要远程登录 Ubuntu，还需要 Ubuntu 开启 SSH 服务。

### 10.1 安装 SSH 服务

在 Ubuntu 上执行：

```bash
sudo apt update
sudo apt install -y openssh-server
```

---

### 10.2 启动 SSH 服务

```bash
sudo systemctl enable ssh
sudo systemctl start ssh
```

查看状态：

```bash
systemctl status ssh
```

如果状态显示：

```text
active (running)
```

说明 SSH 服务正常运行。

---

### 10.3 查看 Ubuntu 用户名

在 Ubuntu 上执行：

```bash
whoami
```

假设输出为：

```text
your-username
```

那么后面 Windows SSH 登录时就使用这个用户名。

---

## 11. Windows 远程 SSH 登录 Ubuntu

在 Windows PowerShell 中执行：

```powershell
ssh Ubuntu用户名@Ubuntu的ZeroTier_IP
```

例如：

```powershell
ssh your-username@10.10.10.23
```

第一次连接时，可能会提示：

```text
Are you sure you want to continue connecting?
```

输入：

```text
yes
```

然后输入 Ubuntu 用户密码，即可登录。

成功后，你会进入实验室 Ubuntu 服务器的命令行环境。

### 12.1 远程登录服务器

```powershell
ssh your-username@10.10.10.23
```

### 12.2 从 Windows 复制文件到 Ubuntu

在 Windows PowerShell 中执行：

```powershell
scp .\test.py your-username@10.10.10.23:/home/your-username/
```

### 12.3 从 Ubuntu 复制文件到 Windows

在 Windows PowerShell 中执行：

```powershell
scp your-username@10.10.10.23:/home/your-username/result.txt .\
```

### 12.4 访问 Ubuntu 上的 Web 服务

如果 Ubuntu 上启动了一个测试 Web 服务：

```bash
python3 -m http.server 8000
```

```text
http://10.10.10.23:8000
```

就可以看到 Ubuntu 当前目录下的文件列表。

## 13. 设备重启后是否还有效

一般情况下，只要 ZeroTier 服务正常开机自启，设备重启后会自动重新加入之前的网络。

Ubuntu 上可以确认 ZeroTier 服务开机自启：

```bash
sudo systemctl is-enabled zerotier-one
```

如果不是 `enabled`，则执行：

```bash
sudo systemctl enable zerotier-one
```

Windows 上 ZeroTier One 通常会作为后台服务运行。只要没有手动退出或卸载，重启后一般会自动恢复连接。

## 14. 常见问题排查

### 14.1 `zerotier-cli status` 不是 ONLINE

检查设备是否能正常联网。

Ubuntu：

```bash
ping 8.8.8.8
ping www.baidu.com
```

Windows：

```powershell
ping 8.8.8.8
ping www.baidu.com
```

如果设备本身不能访问互联网，ZeroTier 也无法正常连接。

---

### 14.2 `listnetworks` 显示 ACCESS_DENIED

这是最常见问题。

原因：

```text
设备已经加入网络，但还没有在 ZeroTier Central 后台授权。
```

解决方法：

```text
进入 ZeroTier Central
→ 打开对应 Network
→ 找到 Members / Member Devices
→ 授权该设备
```

授权后，再执行：

Ubuntu：

```bash
sudo zerotier-cli listnetworks
```

Windows：

```powershell
zerotier-cli listnetworks
```

状态应变为：

```text
OK
```

---

### 14.3 Windows 能加入网络，但 ping 不通 Ubuntu

先确认 Ubuntu 是否已获得 ZeroTier IP：

```bash
sudo zerotier-cli listnetworks
```

然后确认 Windows 使用的是 Ubuntu 的 ZeroTier IP，而不是实验室局域网 IP。

错误示例：

```powershell
ssh your-username@192.168.1.100
```

如果你人在外地，这个实验室局域网 IP 通常不可达。

正确示例：

```powershell
ssh your-username@10.10.10.23
```

---

### 14.4 ping 不通，但 SSH 能连

这通常不是大问题。某些系统或防火墙会禁用 ICMP ping，但 TCP 服务仍可访问。

直接测试 SSH：

```powershell
ssh your-username@10.10.10.23
```

或者测试端口：

```powershell
Test-NetConnection 10.10.10.23 -Port 22
```

如果 `TcpTestSucceeded` 为 `True`，说明 SSH 端口可达。

---

### 14.5 SSH 连接失败：Connection refused

这通常说明网络通了，但 Ubuntu 没有开启 SSH 服务。

在 Ubuntu 上执行：

```bash
sudo apt install -y openssh-server
sudo systemctl enable ssh
sudo systemctl start ssh
systemctl status ssh
```

---

### 14.6 SSH 连接失败：Connection timed out

这通常说明网络没有打通，或者防火墙阻挡了访问。

依次检查：

```bash
sudo zerotier-cli status
sudo zerotier-cli listnetworks
ip addr
```

Ubuntu 防火墙检查：

```bash
sudo ufw status
```

如果启用了 ufw，可以允许 SSH：

```bash
sudo ufw allow ssh
```

---

### 14.7 Windows 找不到 `zerotier-cli`

可以尝试使用 ZeroTier 托盘图标操作，不一定必须用命令行。

如果一定要用命令行，确认：

```text
1. ZeroTier One 已经安装；
2. PowerShell 是管理员权限；
3. ZeroTier 安装目录已经加入 PATH；
4. 或者进入 ZeroTier 安装目录执行 zerotier-cli。
```

---

## 15. 安全建议

### 15.1 不要公开 Network ID

Network ID 本身不是密码，但不建议随意公开。你的网络应该保持 Private，并且所有设备都需要后台授权。

---

### 15.2 只授权自己的设备

看到陌生设备申请加入时，不要授权。

---

### 15.3 给设备命名

建议每台设备都命名清楚：

```text
Windows-Laptop
Lab-Ubuntu-Server
Home-Desktop
UAV-GroundStation
```

这样以后排查问题时不会混乱。

---

### 15.4 Ubuntu 使用强密码或 SSH Key

如果服务器长期远程访问，建议使用 SSH Key 登录，减少密码暴力尝试风险。

生成 SSH Key：

```powershell
ssh-keygen
```

把公钥复制到 Ubuntu：

```powershell
ssh-copy-id your-username@10.10.10.23
```

Windows 没有 `ssh-copy-id` 时，可以手动把公钥内容追加到 Ubuntu 的：

```bash
~/.ssh/authorized_keys
```

## 16. 建议的最小验证流程

第一次学习 ZeroTier 时，建议严格按这个顺序来：

```text
1. 创建 ZeroTier Network；
2. Windows 安装 ZeroTier；
3. Windows 加入 Network；
4. Ubuntu 安装 ZeroTier；
5. Ubuntu 加入 Network；
6. ZeroTier 后台授权两台设备；
7. 记录 Ubuntu 的 ZeroTier IP；
8. Windows ping Ubuntu 的 ZeroTier IP；
9. Ubuntu 开启 SSH；
10. Windows 执行 ssh 用户名@Ubuntu-ZeroTier-IP；
11. 成功远程登录。
```

最小可用命令如下。

Ubuntu 服务器：

```bash
curl -s https://install.zerotier.com | sudo bash
sudo zerotier-cli join 你的Network_ID
sudo zerotier-cli status
sudo zerotier-cli listnetworks

sudo apt update
sudo apt install -y openssh-server
sudo systemctl enable ssh
sudo systemctl start ssh
```

Windows 笔记本：

```powershell
zerotier-cli join 你的Network_ID
zerotier-cli status
zerotier-cli listnetworks
ping Ubuntu的ZeroTier_IP
ssh Ubuntu用户名@Ubuntu的ZeroTier_IP
```

## 17. 当前场景的最终建议

对于“外地笔记本远程实验室服务器”的需求，先不要配置复杂的局域网路由，也不要租用阿里云服务器。

建议先跑通：

```text
Windows 笔记本
    ↓ ZeroTier
Ubuntu 实验室服务器
    ↓ SSH
远程登录服务器
```

等这个闭环稳定后，如果后续还想通过实验室 Ubuntu 服务器访问同一实验室局域网里的 MaixCAM、RK3588、NAS、摄像头等设备，再进一步学习 ZeroTier 的 Managed Routes、IP Forwarding、NAT 网关等高级配置。当前阶段不用一开始就做这些。
