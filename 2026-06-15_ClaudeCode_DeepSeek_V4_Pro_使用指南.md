## 2. 基础交互：提问与对话

### 2.1 直接提问

在命令行中直接输入问题，Claude Code 会结合你的项目上下文回答：

```bash
# 通用编程问题
> Python中asyncio和threading有什么区别？什么时候用哪个？

# 项目相关的问题
> 这个项目的认证逻辑是怎么实现的？

# 代码生成
> 帮我写一个Python脚本，递归遍历目录找出所有超过100MB的文件
```

### 2.2 多轮对话与上下文

Claude Code 会保持整个会话的上下文。你可以：

```bash
> 帮我分析src/auth.py的登录逻辑
> 有什么安全问题吗？
> 帮我修复这些问题，并加上注释说明
> 顺便写个单元测试
```

每一轮都会基于之前的所有对话内容来理解和执行。

### 2.3 斜杠命令

Claude Code 内置了许多斜杠命令，输入 `/` 即可看到列表：

| 命令 | 功能 |
|------|------|

| `/clear` | 清空当前对话 |
| `/compact` | 压缩对话上下文（节省Token） |
| `/config` | 修改配置（主题、模型等） |
| `/model` | 切换模型 |

## 4. 代码编辑与重构

### 4.1 精确编辑（Edit）

Claude Code 使用 `old_string → new_string` 的精确替换模式，安全且可控：

```bash
> 把src/utils.py中的get_config函数名改为load_config，同时更新所有调用处

> 在src/models.py第45行之后添加一个email字段

> 把app.js中所有的var声明替换为const/let（根据是否被重新赋值来决定）
```

### 4.2 整文件重写（Write）

当需要大幅修改时：

```bash
> 根据以下需求重写src/api/handlers.py：
  - 添加请求参数校验
  - 添加统一的错误处理
  - 添加响应日志记录
```

### 4.3 跨文件重构

```bash
# 大规模重构
> 把项目从JavaScript迁移到TypeScript：
  1. 先把所有.js文件重命名为.ts
  2. 添加类型注解
  3. 更新tsconfig.json
  4. 更新package.json的构建脚本

# 提取公共逻辑
> 在src/components/中找出所有重复的表单验证逻辑，
  提取到src/hooks/useFormValidation.ts中
```

## 9. 计划模式：架构设计

在执行复杂改动前，你可以要求 Claude 先制定计划：

```bash
> 我想把现有的REST API迁移到GraphQL，帮我做个迁移计划和风险评估
```

1. **探索代码库**：理解现有架构

5. **等待审批**：你审查计划后再动手实现

## 10. 多智能体协作与工作流

这是 Claude Code 最强大的功能之一。你可以同时启动多个子Agent并行处理任务。

### 10.1 并行代码审查

```bash
> 用3个Agent并行审查src/目录的代码：
  - Agent 1: 检查安全问题
  - Agent 2: 检查性能问题
  - Agent 3: 检查代码风格和最佳实践
  最后汇总所有发现
```

### 10.2 Workflow工作流编排

对于大规模任务，可以使用Workflow进行系统化的多阶段处理：

**代码审查工作流示例**：

- **阶段1 - 扫描**：并行搜索所有可疑模式（SQL注入、XSS、硬编码密钥...）
- **阶段2 - 验证**：对每个发现的疑似问题，派出独立Agent进行对抗性验证
- **阶段3 - 修复**：对确认的问题逐一修复

**迁移工作流示例**：

- **阶段1 - 发现**：并行扫描所有需要迁移的文件
- **阶段2 - 迁移**：每个文件在独立的工作树中迁移（避免相互干扰）
- **阶段3 - 验证**：运行测试、对比输出，确保迁移正确

### 10.3 使用场景

```bash
# 安全审计（使用ultracode模式进行深度审查）
> 对这个项目进行全面的安全审计

# 大规模代码迁移
> 用workflow把整个项目从moment.js迁移到day.js

# 并行研究
> 同时研究3个不同的状态管理方案（Redux/Zustand/Jotai），
  从性能、学习曲线、社区生态三个维度比较，推荐最合适的
```

## 14. 配置与个性化

### 14.1 settings.json

你的配置文件位于 `~/.claude/settings.json`：

```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "sk-...",
    "ANTHROPIC_BASE_URL": "https://api.deepseek.com/anthropic",
    "ANTHROPIC_MODEL": "deepseek-v4-pro"
  },
  "effortLevel": "high",
  "theme": "dark",
  "model": "deepseek-v4-pro"
}
```

### 14.2 Hooks（事件钩子）

你可以配置在特定事件发生时自动执行操作：

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "prettier --write $CLAUDE_TOOL_USE_FILE_PATH"
          }
        ]
      }
    ]
  }
}
```

常用Hook场景：
- **每次编辑后自动格式化**代码
- **提交前运行lint和测试**
- **会话开始时自动拉取最新代码**
- **会话结束时自动清理临时文件**

### 14.3 自定义斜杠命令

在 `~/.claude/commands/` 目录下创建Markdown文件：

```markdown
# ~/.claude/commands/deploy.md
帮我把当前分支的代码部署到生产环境：
1. 确认当前在main分支
2. 运行全部测试
3. 构建Docker镜像
4. 推送到镜像仓库
5. 触发K8s滚动更新
6. 监控部署状态直到完成
```

然后使用 `/deploy` 一键触发。

### 14.4 键盘快捷键

通过 `/keybindings-help` 或直接编辑 `~/.claude/keybindings.json` 来自定义快捷键。

```markdown
# CLAUDE.md for MyProject

## 项目概述
这是一个电商后端服务，使用FastAPI + PostgreSQL。

## 技术栈
- Python 3.12, FastAPI, SQLAlchemy 2.0
- PostgreSQL 16, Redis 7
- Docker + Docker Compose

## 代码规范
- 使用black格式化，行宽100
- 类型注解是必须的
- 所有public函数必须有docstring

## 关键约定
- API响应格式统一使用{"code": int, "data": ..., "message": str}

- 环境变量通过.env文件管理，不要把敏感信息提交到git

- 使用pytest，测试文件放在tests/目录
- 运行测试前需要启动docker-compose的测试数据库
```
